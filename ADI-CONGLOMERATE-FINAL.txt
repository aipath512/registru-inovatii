Issued by: AiVenture S.R.L. / 5thElement.ai

DEFINITION:
AI Delivery Infrastructure (ADI) is a decoupled architectural model where a website's intelligence layer, semantic metadata, and machine-readable artifacts are managed externally and delivered on-demand via edge computing or dynamic injection. It transitions the web from static document display to a dynamic, contract-based intelligence access point.

THE MANIFESTO:
1. EXTERNALIZATION: Intelligence must not live inside the CMS. It exists as a centralized, versioned, and governed layer outside the host.
2. DUAL-RAIL EXECUTION: A single AI package delivered via two paths: Direct Edge Delivery (Cloudflare Workers) for zero-friction control, or Origin-Assisted Injection for legacy environments.
3. CONTROLLED POROSITY: The website acts as an intelligent membrane. It delivers a high-end experience to humans while exposing a supplemental, authorized semantic layer to AI agents.
4. SOVEREIGNTY: Value shifts from owning code to possessing the Control Plane of intelligence. The site is a controlled endpoint, not a final container.

ADI CORE PRINCIPLES:
I.   Separation       — Website execution and AI execution are independent layers.
II.  Externalization  — Intelligence lives in the infrastructure, not inside the website.
III. Dynamic Delivery — AI is delivered on demand, not stored statically.
IV.  Dual-Rail        — The same AI resources are deliverable through multiple execution paths.
V.   Contextual Access— Delivery is governed by identity, policies, context and time.
VI.  Porosity         — The website becomes a permeable membrane, not a closed container.

CONCLUSION:
ADI is not a tool or a plugin. It is a new infrastructure layer of the web — external, centralized, contextual, and reusable across any platform.

ADI serves as the canonical registry for all other modules, utilizing SHA-256 for artifact identification and OpenTimestamps (OTS) for temporal anchoring, ensuring information sovereignty.

ONE-LINER:
The site accesses AI. AI does not live inside the site.
